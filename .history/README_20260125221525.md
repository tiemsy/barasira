# Barasira

Barasira est une plateforme de mise en relation entre clients et prestataires de services, construite avec Laravel. Elle expose une API REST (protégée par Sanctum) et un front moderne (Vue/Inertia).

## Présentation
- Objectif: permettre aux clients de publier des missions et aux prestataires d’y postuler, exécuter, puis recevoir une évaluation.
- Domaines couverts: utilisateurs, services, catégories, missions, candidatures, avis, CV/portfolio.

## Architecture
- Backend: Laravel (PHP 8.1+)
- Authentification: Laravel Sanctum (cookies ou tokens)
- Front: Vue 3 + Inertia.js (selon configuration du repo)
- Build: Vite
- Documentation API: L5-Swagger (génération) + openapi.yaml minimal (manuel) à la racine

## Démarrage rapide
bash

1) Dépendances
composer install
npm install

2) Env et clé
cp .env.example .env
php artisan key:generate

3) Base de données
php artisan migrate --seed

4) Serveurs dev
php artisan serve
npm run dev

(optionnel) Mail local
maildev

## Variables d’environnement
- DB_*: connexion base de données
- SANCTUM_STATEFUL_DOMAINS, SESSION_DOMAIN: pour SPA/CORS
- APP_URL, FRONTEND_URL: liens applicatifs
- MAIL_*: envoi d’e-mails (sandbox en dev)

## Commandes utiles
bash

Vérifier les routes API
php artisan route:list --path=api

Générer la doc Swagger (L5-Swagger)
php artisan l5-swagger:generate

Cache/optimize (prod)
php artisan config:cache
php artisan route:cache
php artisan view:cache

## Documentation API
- Spécification générée (L5-Swagger): storage/api-docs/api-docs.yaml
- Spec minimal manuelle: openapi.yaml (à la racine)
- Routes principales (routes/api.php):
  - Auth: POST /api/login, POST /api/register, POST /api/logout, GET /api/me
  - Ressources: users, user-skills, service-categories, services (+ GET /api/services-search), missions, reviews, resumes, portfolio-items

## Modules principaux
- Users, Profiles (skills, resumes, portfolio)
- Services & Categories
- Missions (création par clients)
- Reviews (avis/notations)
- Recherche de services (services-search)

## Flux “Mission” recommandé
1) Client crée une mission (POST /api/missions → status: pending)
2) Prestataire postule (endpoint applications si présent, sinon action dédiée)
3) Client accepte une candidature (mission → in_progress)
4) Prestataire exécute et clôture (PATCH /api/missions/{id} → completed)
5) Client dépose un avis (POST /api/reviews)

États:
- Mission: pending → in_progress → completed (→ cancelled si besoin)

## Qualité, tests et style
- Tests: PHPUnit
- Style: Laravel Pint (recommandé)
- Debug: Laravel Debugbar (dev), Ignition pour erreurs
- Recommandations:
  - FormRequest pour la validation
  - Policies pour l’accès (missions, reviews…)
  - Resources (JsonResource) pour les réponses
  - Pagination par défaut

## Déploiement
- Build assets: `npm run build`

- Vérifier CORS/Sanctum (SESSION_DOMAIN, SANCTUM_STATEFUL_DOMAINS)
